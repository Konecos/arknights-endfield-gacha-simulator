# 明日方舟：终末地 全面测试 特许寻访寻访模拟器

该项目提供了一个高度可定制的 Web 模拟器，用于分析《明日方舟：终末地》中特许寻访（抽卡）机制的概率分布和累积概率。用户可以自由修改游戏中的各项概率和保底参数，以进行准确的抽卡成本估算和风险分析。

**项目地址:** https://github.com/Konecos/arknights-endfield-gacha-simulator

## ✨ 主要功能

* **参数配置**：所有核心概率和保底机制（包括基础概率、保底起始、提升幅度、硬保底、特许必得保底等）均可通过界面自定义，避免硬编码。

* **抽卡模拟**：基于配置的参数，进行大规模（数十万次）的单次出货模拟。

* **概率密度图 (Bar Chart)**：直观展示单次获得 UP 干员所需抽数的分布情况，并标记平均值和必得线。

* **累积概率图 (CDF)**：展示在特定抽数内（含）获得 UP 干员的总累积概率，帮助用户规划预算和确定稳妥抽数。

* **关键数据统计**：提供平均出货抽数、总消耗抽数、特许必得保底触发率等核心指标。

## ⚙️ 参数说明

以下是模拟器中可以自定义的核心参数：

| **参数名称** | **描述** | **默认值** | 
| ----- | ----- | ----- | 
| **6星基础概率 (%)** | 获得任意 6 星角色的基础概率。 | 0.8 | 
| **5星基础概率 (%)** | 获得任意 5 星角色的基础概率。 | 8.0 | 
| **概率提升起始 (抽)** | 6 星概率开始提升的抽数。 | 65 | 
| **每抽提升幅度 (%)** | 超过起始抽数后，每抽增加的 6 星概率百分比。 | 5.0 | 
| **6星硬保底 (抽)** | 保证必定获得 6 星的抽数上限（通常为 99%）。 | 80 | 
| **5星保底 (抽)** | 保证必定获得 5 星的抽数上限。 | 10 | 
| **特许必得保底 (抽)** | 特许机制中，用于保证获取 UP 干员的抽数（如 120 抽）。设为 0 可关闭。 | 120 | 
| **模拟总抽数上限** | 整个模拟过程消耗的总抽数，用于控制模拟规模和精度。 | 500,000 | 

## 🚀 运行与使用

该项目基于 React 和 Tailwind CSS 构建，使用 `recharts` 进行图表展示。

### 💻 本地运行

1. **克隆仓库:**

   ```
   git clone https://github.com/Konecos/arknights-endfield-gacha-simulator.git
   cd arknights-endfield-gacha-simulator
   ```

2. **安装依赖:**

   ```
   npm install
   # 或 yarn install
   ```

3. **启动应用:**

   ```
   npm start
   # 或 yarn start
   ```

   应用将在 `http://localhost:3000` 启动。

### 📊 使用指南

1. **配置参数**：在顶部的“参数设置”面板中，输入你想要模拟的概率和保底数值。

2. **运行模拟**：点击“运行模拟”按钮，程序将根据你的配置进行大规模的概率模拟。

3. **查看结果**：结果区域将展示平均出货数、必得触发率以及两个关键图表：

   * **分布图**：查看获得 UP 角色最有可能发生在哪个抽数范围。

   * **累积概率图**：查看你需要多少抽才能有 50%、90% 或 99% 的概率获取 UP 角色。
